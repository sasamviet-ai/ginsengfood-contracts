## Boundary Locks

- External `/v1` consumer APIs stay read/check oriented unless a source document names a mutation boundary.
- Admin/internal routes may exist for ops-core owned operational workflows, but they must still cite exact source paths.
- AsyncAPI broker, topic, retry, and outbox details remain future integration decisions until approved by source docs or owners.
- Examples and fixtures are generated after schema/enums settle; they must not introduce fields that are not source-backed.
- IVR contract files may cite Phase 8 SRS paths only when the path is listed in `docs/source-map.md`; Order Core remains the only owner that can transition order state after an IVR result signal.
